  <files>src/lib/chat/persistence.ts, src/lib/chat/rate-limit.ts</files>
  <action>
1. Criar `src/lib/chat/persistence.ts`:

   a) `saveMessages(supabase, userId, messages)`:
      - Recebe o client Supabase, userId e array de messages no formato Vercel AI SDK
      - Filtra apenas mensagens com role 'user' ou 'assistant' (ignora 'system', 'data')
      - Para cada mensagem, faz upsert na tabela `messages`:
        - `id`: usar o message.id do Vercel AI SDK como identificador
        - `user_id`: userId
        - `role`: message.role
        - `content`: message.content
        - `tool_invocations`: JSON.stringify(message.toolInvocations) se existir, null caso contrario
        - `created_at`: usar message.createdAt se disponivel, senao NOW()
      - Usar `upsert` com `onConflict: 'id'` para evitar duplicatas em re-envios
      - NAO lancar excecao em caso de erro — logar com console.error e retornar silenciosamente (persistencia e best-effort, nao deve quebrar o chat)

   b) `loadMessages(supabase, userId, limit = 50)`:
      - Busca mensagens do usuario ordenadas por created_at ASC
      - Limita ao `limit` mais recentes (ORDER BY created_at DESC LIMIT limit, depois reverter)
      - Retorna no formato compativel com Vercel AI SDK Message[]
      - Reconstroi toolInvocations do JSON se existir

2. Criar `src/lib/chat/rate-limit.ts`:

   a) `checkRateLimit(supabase, userId)`:
      - Calcula yearMonth: `new Date().toISOString().slice(0, 7)` (ex: '2026-04')
      - Chama `supabase.rpc('check_and_increment_usage', { p_user_id: userId, p_year_month: yearMonth })`
      - Retorna o resultado tipado: `{ allowed: boolean, current: number, limit: number, plan: string }`
      - Se RPC falhar (erro de rede, funcao nao existe), retornar `{ allowed: true, current: 0, limit: 0, plan: 'unknown' }` — fail-open para nao bloquear o chat por erro de infra. Logar o erro com console.error.

IMPORTANTE: Ambos os modulos devem ser importaveis em Route Handlers (server-side only). NAO adicionar 'use client'.

     const supabase = createClient()

3. Modificar `POST` handler:

   a) APOS autenticacao e ANTES de streamText, adicionar rate limiting:
      ```
      const rateLimit = await checkRateLimit(supabase, user.id)
      if (!rateLimit.allowed) {
        return Response.json({
          error: `Limite de ${rateLimit.limit} mensagens/mes atingido. Seu plano: ${rateLimit.plan}.`,
          code: 'RATE_LIMITED',
          current: rateLimit.current,
          limit: rateLimit.limit,
        }, { status: 429 })
      }
      ```

   b) JANELA DE CONTEXTO (RF-02.8): Antes de passar messages ao streamText, limitar a 20 mensagens:
      ```
      const contextMessages = messages.slice(-20)
      ```
      Passar `contextMessages` (nao `messages`) para streamText. As mensagens completas ainda sao salvas no banco.

   c) Adicionar `onFinish` callback ao streamText para salvar mensagens APOS stream completo:
      ```
      const result = streamText({
        model: anthropic(ANJA_MODEL),
        system: ANJA_SYSTEM_PROMPT,
        messages: contextMessages,
        tools: anjaTools,
        maxSteps: ANJA_MAX_STEPS,
        maxTokens: ANJA_MAX_TOKENS,
        onFinish: async ({ text, toolCalls, toolResults }) => {
          // Salvar a mensagem do usuario (ultima do array messages) e a resposta da Anja
          // Best-effort — nao bloqueia o stream
          try {
            const assistantMessage = {
              id: crypto.randomUUID(),
              role: 'assistant' as const,
              content: text,
              toolInvocations: toolCalls?.map((tc, i) => ({
                toolCallId: tc.toolCallId,
                toolName: tc.toolName,
                args: tc.args,
                state: 'result' as const,
                result: toolResults?.[i]?.result,
              })),
            }
            const userMessage = messages[messages.length - 1]
            await saveMessages(supabase, user.id, [userMessage, assistantMessage])
          } catch (e) {
            console.error('Erro ao salvar mensagens:', e)
          }
        },
      })
      ```

   d) Retornar `result.toDataStreamResponse()` como antes.

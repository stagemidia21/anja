  <action>
1. Criar `src/lib/tools/types.ts`:
   - Interface `ToolResult<T = unknown>` com campos: `success: boolean`, `data?: T`, `error?: string`
   - Type helper `ToolSuccess<T>` = `{ success: true, data: T }`
   - Type helper `ToolError` = `{ success: false, error: string }`

2. Criar `src/lib/tools/index.ts`:
   - Importar `tool` de `'ai'` e `z` de `'zod'`
   - Exportar `anjaTools` como objeto com as seguintes tools definidas via `tool()`:

   a) `create_calendar_event`:
      - description: "Cria um evento no Google Calendar do usuario"
      - parameters: z.object com title (string), start_datetime (string ISO 8601), end_datetime (string ISO 8601), description (string optional), location (string optional)
      - execute: stub que retorna `{ success: false, error: 'Google Calendar ainda nao conectado. Disponivel na proxima atualizacao.' }`

   b) `list_calendar_events`:
      - description: "Lista eventos do Google Calendar em um periodo"
      - parameters: z.object com start_date (string YYYY-MM-DD), end_date (string YYYY-MM-DD)
      - execute: stub que retorna `{ success: false, error: 'Google Calendar ainda nao conectado. Disponivel na proxima atualizacao.' }`

   c) `create_task`:
      - description: "Cria uma nova tarefa para o usuario"
      - parameters: z.object com title (string), priority (enum alta/media/baixa, default media), category (string optional), due_date (string optional YYYY-MM-DD), note (string optional)
      - execute: stub que retorna `{ success: false, error: 'Gestao de tarefas disponivel na proxima atualizacao.' }`

   d) `list_tasks`:
      - description: "Lista as tarefas do usuario"
      - parameters: z.object com status (enum fazer/fazendo/feito, optional), priority (enum alta/media/baixa, optional)
      - execute: stub que retorna `{ success: false, error: 'Gestao de tarefas disponivel na proxima atualizacao.' }`

CRITICO (per RF-02.6): Nenhuma funcao `execute` deve JAMAIS lancar excecao (throw). Todas devem retornar `{ success: false, error: string }` em caso de erro. Envolver cada execute em try/catch que retorna ToolResult em caso de erro inesperado.

Cada execute deve ter a assinatura `async (args) => { try { ... } catch (e) { return { success: false, error: 'Erro inesperado.' } } }`.

Criar `src/app/api/chat/route.ts` com:

   - `anthropic` de `@ai-sdk/anthropic`
   - `streamText` de `ai`
   - `createClient` de `@/lib/supabase/server`
   - `ANJA_SYSTEM_PROMPT, ANJA_MODEL, ANJA_MAX_TOKENS, ANJA_MAX_STEPS` de `@/lib/anthropic`
   - `anjaTools` de `@/lib/tools`

2. Exportar `export const maxDuration = 60` (para evitar timeout da Vercel em streaming longo)

3. Exportar `async function POST(req: Request)`:

   a) Autenticacao server-side:
      ```
      const supabase = createClient()
      const { data: { user } } = await supabase.auth.getUser()
      if (!user) return Response.json({ error: 'Nao autenticado' }, { status: 401 })
      ```

   b) Extrair messages do body:
      ```
      const { messages } = await req.json()
      ```

   c) Chamar streamText:
      ```
      const result = streamText({
        model: anthropic(ANJA_MODEL),
        system: ANJA_SYSTEM_PROMPT,
        messages,
        tools: anjaTools,
        maxSteps: ANJA_MAX_STEPS,
        maxTokens: ANJA_MAX_TOKENS,
      })
      ```
      NOTA: `streamText` NAO precisa de await — retorna um objeto com `.toDataStreamResponse()`.

   d) Retornar stream:
      ```
      return result.toDataStreamResponse()
      ```

4. NAO implementar rate limiting ou persistencia de mensagens neste plan — isso vem no plan 02-04 que modifica este arquivo.

5. NAO usar `export const runtime = 'edge'` — manter Node.js runtime para compatibilidade com cookies() do Supabase.
